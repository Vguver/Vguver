# Hi, I'm Vguver

I developed an interest in computing and electronics at an early age. Although life eventually took me into a different branch of engineering, Linux brought me back to that original curiosity.

After spending a lot of time learning through experimentation, testing different setups, and studying how other people build their systems, I started creating practical tools that help others explore Linux with more confidence.

This GitHub is where I share projects designed to make Linux easier to understand, easier to experiment with, and easier to adapt. A big part of that motivation comes from helping family members learn, replicate useful setups, and shape their computers around what they actually want.

My background is not in software engineering, so my work is shaped by hands-on learning, experimentation, and careful observation. I study how others structure their desktops, workflows, and dotfiles, then turn those ideas into cleaner, more reusable, and more approachable tools.

## What I build
- Practical Linux tools for experimentation and learning
- Workflows that make desktop customization easier to understand
- Reusable setups that help others transition into Linux
- Projects inspired by real-world use, teaching, and hands-on testing

## Featured project — isolated-desktops

[isolated-desktops](https://github.com/Vguver/isolated-desktops) is a session-profile manager for testing multiple Linux desktop setups on a single machine while keeping separate session homes and a cleaner workflow for analysis, installation, verification, and iteration.

It is built for people who want to:
- compare desktop environments without mixing configuration
- understand how other Linux setups are structured
- experiment more safely before committing to a workflow
- learn by editing, testing, and refining real profiles

![isolated-desktops status](./assets/isolated-desktops-status.png)

## Why this matters to me

A lot of this started with a simple goal: replicating my desktop across different computers and explaining Linux to younger family members in a way that felt practical, not overwhelming. That is still the direction behind my work today.

## Current focus

I am currently focused on building Linux tools that help people test, understand, and customize their desktop environments in a more approachable way.
